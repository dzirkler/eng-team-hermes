## THE open question (task #24 in the Cowork task tracker — not visible to Claude Code, repeated here)

**Does a Kanban "worker profile" = a separate Hermes profile
(`hermes profile create <name>`, its own `HERMES_HOME`, own `config.yaml`,
own `SOUL.md`, own skills dir)? Or is a Kanban "worker" something else —
e.g. a lighter-weight role/label that doesn't require a full separate
profile per specialist?**

Why it matters: the entire migration plan's guardrail design assumes a
per-worker-profile `toolsets:` restriction ("orchestrator's toolset simply
doesn't include edit/write — enforceable by construction"). **No such
config key was found anywhere in two full doc-page reads** (CLI reference,
Kanban feature page) — only a session-scoped `-t/--toolsets <csv>` flag on
`hermes chat`, which doesn't map onto a persistent per-worker restriction.
If toolset restriction isn't actually a real mechanism, the guardrail
design needs to lean entirely on `pre_tool_call` hooks instead (which ARE
confirmed real) — but that's a different, more code-heavy design than what
`profiles/*/profile.yaml` currently encodes.
